# INBOX — задача для Claude

**ID:** T-009 (Pilot Demo)
**Дата:** 2026-07-06
**Автор (PM):** ChatGPT

## Цель
Довести демо до вида готовой РЕГИОНАЛЬНОЙ информационной системы, максимально убедительно
показывающей ценность КУРАТОРА заказчику (не «техническое демо»).

## Что сделать (по приоритету PM)
1. **Реалистичные данные** — 30–50 мер поддержки Краснодарского края, 10–15 жизненных ситуаций,
   реальные документы/основания/требования (заказчик узнаёт знакомые меры → доверие).
2. **White Label** — название региона, логотип/герб (заменяемый), цветовая тема, экран
   приветствия, смена региона через настройки администратора без кода.
3. **Экран истории сопровождения (Case Timeline)** — хронология: дата, событие, исполнитель,
   текущий статус; обновляется автоматически по действиям гражданина и координатора.
4. **Руководительский дашборд (MVP)** — новых кейсов, активных заявок, завершённых кейсов,
   распределение заявок по статусам, топ-5 самых востребованных мер.
5. **Сценарий демонстрации** — кто демонстрирует, что говорит на каждом шаге, какие ценности
   акцентирует, ответы на вероятные вопросы заказчика. Цель — любой член команды проводит
   одинаково сильную демонстрацию за 10–12 минут.

## Критерий готовности
Система наполнена реалистичными данными КК; интерфейс выглядит как региональная система;
реализован Case Timeline; есть управленческий дашборд; подготовлен воспроизводимый сценарий демо.

## Позиционирование (рекомендация PM)
Начинать демонстрацию не с «мы сделали систему льгот», а с боли: «сегодня участник СВО
вынужден сам искать десятки мер поддержки…» → показать, как КУРАТОР это решает.
