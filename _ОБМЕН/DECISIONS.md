# DECISIONS — решения основателя по проекту «КУРАТОР»

Персистентный журнал ключевых решений Артура (основатель). Приоритет над допущениями,
которые Claude/ChatGPT приняли по умолчанию. Обновляется по мере новых решений.

## 2026-07-06

**D-1. Целевая категория MVP.** Стартуем с участников СВО и семей. Позже — расширение на
инвалидов и иные социально уязвимые группы. (Архитектура ядра не переделывается — новая
категория = новый набор правил/шаблонов.)

**D-2. Пилотный регион.** Краснодарский край — подтверждён.

**D-3. Метрики успеха пилота.**
- Число поданных заявок.
- Удовлетворённость (граждан и координаторов).
- Передача успешной истории от региона к региону — среди министерств и координаторов.
- Уточнение PM (v1.2): вместо «передачи истории» измеримее формулировать как
  **повторяемость внедрения** — успешный пилот в КК → готовность подключить второй регион
  без изменения кода → положительная рекомендация первого заказчика для следующего.

**D-6. Философия MVP (уточнено PM, v1.2).** Сейчас строим не промышленную систему, а
**инструмент, чтобы продать проект государству**. Успех первого MVP = заказчик поверил, что
система изменит сопровождение участников СВО. Приоритет — ОДИН безупречно работающий сквозной
сценарий, а не покрытие всего процесса. KPI сместить с «числа реализованных функций» на
качество демонстрации и успешность пилота. Роадмап — ~13 недель для команды из 2 человек при
активном использовании ИИ (ИИ = «третий член команды»).

**D-4. Цель этапа и команда.** Основная цель — донести ценность до заказчика (госсектор).
НО код нужно делать так, чтобы можно было продемонстрировать рабочий прототип (demo-able).
Команда: 2 человека (Артур + ещё один). → Scope MVP держать максимально узким; приоритет
P0-фич и демонстрируемого сценария «вход → анкета → подбор → объяснение → заявка».

**D-5. Процесс.** Автономный цикл ChatGPT (PM) ↔ Claude (исполнитель) продолжаем в текущем
режиме. Саммари для нового чата не требуется — Артур покупает платную версию ChatGPT
(лимит контекста снимается).
