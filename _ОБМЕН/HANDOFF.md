# HANDOFF — протокол связи ChatGPT ↔ Claude

Этот файл — единая инструкция для обмена задачами в проекте «КУРАТОР».
Папка `_ОБМЕН` — общая доска. Через неё передаются задачи и результаты.

## Роли

| ChatGPT — постановщик | Claude — исполнитель |
|---|---|
| Продукт-менеджмент: приоритеты, ТЗ, user stories | Инженерия: код, схемы БД, API, архитектура |
| Маркетинг: позиционирование, тексты, стратегия | Технические документы, диаграммы, файлы |
| Приёмка результата с точки зрения продукта | Реализация и доставка результата в папку |

## Как это работает

1. **ChatGPT ставит задачу.** Артур копирует ТЗ от ChatGPT и сохраняет его в файл
   `_ОБМЕН/INBOX_task.md` (перезаписывает шаблон своим текстом).
2. **Claude выполняет.** Артур в этом чате пишет «проверь INBOX» (или «выполни задачу»).
   Claude читает `INBOX_task.md`, выполняет, кладёт результат в `_ОБМЕН/OUTBOX_result.md`
   и сами артефакты (код/документы) — в нужные папки проекта.
3. **Приёмка.** Артур показывает `OUTBOX_result.md` ChatGPT. Тот оценивает и ставит
   следующую задачу → возврат к шагу 1.
4. **Журнал.** Каждая завершённая задача одной строкой дописывается в `_ОБМЕН/LOG.md`.

## Правила

- Один INBOX = одна задача (или один связный пакет). Не смешивать несколько тем.
- В INBOX всегда должна быть цель и критерий готовности — иначе Claude уточнит.
- Claude не переписывает INBOX, только читает. Пишет всегда в OUTBOX и артефакты.
- OUTBOX всегда содержит: что сделано, где лежат файлы, что осталось / вопросы к PM.
- Источник правды по проекту — документы в корне папки «Куратор» (архитектура, положения).

## Формат ID задач

`T-001`, `T-002`, ... — сквозная нумерация. ChatGPT ставит ID в INBOX, Claude повторяет
его в OUTBOX и LOG, чтобы ничего не терялось между сессиями.
