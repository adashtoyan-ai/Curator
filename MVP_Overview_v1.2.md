## 1. Что такое «КУРАТОР» (в одном абзаце)

КУРАТОР — платформа цифрового сопровождения граждан. Не каталог льгот и не портал госуслуг, а
система, которая сама знает, кто пользователь и что ему положено, и ведёт его по персональному
маршруту, отвечая каждый день на вопрос «что мне сделать сегодня». Первый сценарий —
сопровождение участников СВО и членов их семей. Архитектура универсальна: новые категории
граждан добавляются как конфигурация правил, без переработки ядра.

---

## 2. Философия MVP v1.2 (главная корректировка)

**Сейчас строится не промышленная система, а инструмент, чтобы донести ценность проекта до
государственного заказчика.**

Первый MVP отвечает на один вопрос заказчика: *«Верит ли он, что эта система изменит
сопровождение участников СВО?»* Если да — MVP успешен.

Из этого следуют приоритеты:
- Фокус — ОДИН безупречно работающий сквозной сценарий, а не покрытие всего процесса.
- KPI смещён с «числа реализованных функций» на **качество демонстрации и успешность пилота**.
- Архитектура — максимально простая (модульный монолит), без преждевременного усложнения.
- ИИ используется как «третий член команды» (двое людей + активная генерация кода/документов).

**Демонстрируемый сквозной сценарий (сердце MVP):**
Вход → Создать кейс → Заполнить анкету → Получить рекомендации → Понять «почему именно эти
меры» → Подать заявку → Координатор её увидел.

## 5. Архитектура в двух словах (v1.2)

Модульный монолит: Web UI → API Gateway → ядро модулей (Auth, Projects, Questionnaire, Rule
Engine/Matching, Applications, Admin — минимальный набор под демо; Documents, Notifications,
Dashboard подключаются по мере надобности) → PostgreSQL (+ Object Storage для файлов). Внешние
провайдеры (SMS, в будущем ЕСИА/госсистемы) — через адаптеры, ядро от них не зависит.
Соответствие 152-ФЗ заложено архитектурно (данные в РФ, шифрование, RBAC, аудит).

---

## 6. Роадмап (v1.2, ~13 недель)

- **Sprint 0** (~1–2 нед) — фундамент: каркас Backend/Frontend, схема БД, CI/CD.
- **Sprint 1** (3 нед) — вход и анкета: SMS, профиль, кейс, life_event, базовая + динамическая анкета.
- **Sprint 2** (3 нед) — ядро ценности: Rule Engine, импорт мер КК, подбор, Explainability.
- **Sprint 3** (3 нед) — документы, заявки, кабинет координатора.
- **Sprint 4** (2 нед) — полировка, обработка ошибок, сборка демо, запуск пилота.

Итог: демонстрируемый сквозной сценарий на реальных льготах Краснодарского края.
